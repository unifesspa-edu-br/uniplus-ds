Cursor-based / keyset pagination. A API retorna apenas hasNext / hasPrev + nextCursor / prevCursor, não o total. Por isso a UI mostra:
  • Anterior + Próximo (disabled nos extremos)
  • "Página N" (contador no client) + "N resultados nesta página"
  • Nunca "de M" ou números saltadores — implicaria total conhecido
Padrão usado por GitHub, Stripe, Twitter, etc. Otimização crítica em tabelas com 1M+ linhas.
